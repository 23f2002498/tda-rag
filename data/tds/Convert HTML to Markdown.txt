<article class="markdown-section" id="main"><h2 id="converting-html-to-markdown"><a class="anchor" data-id="converting-html-to-markdown" href="#/convert-html-to-markdown?id=converting-html-to-markdown"><span>Converting HTML to Markdown</span></a></h2><p>When working with web content, converting HTML files to plain text or Markdown is a common requirement for content extraction, analysis, and preservation. For example:</p><ul><li><strong>Content analysis</strong>: Extract clean text from HTML for natural language processing</li><li><strong>Data mining</strong>: Strip formatting to focus on the actual content</li><li><strong>Offline reading</strong>: Convert web pages to readable formats for e-readers or offline consumption</li><li><strong>Content migration</strong>: Move content between different CMS platforms</li><li><strong>SEO analysis</strong>: Extract headings, content structure, and text for optimization</li><li><strong>Archive creation</strong>: Store web content in more compact, preservation-friendly formats</li><li><strong>Accessibility</strong>: Convert content to formats that work better with screen readers</li></ul><p>This tutorial covers both converting existing HTML files and combining web crawling with HTML-to-text conversion in a single workflow – all using the command line.</p><h3 id="defuddle-cli"><a class="anchor" data-id="defuddle-cli" href="#/convert-html-to-markdown?id=defuddle-cli"><span>defuddle-cli</span></a></h3><p><a href="https://github.com/defuddle/defuddle" rel="noopener" target="_blank">defuddle-cli</a> specializes in HTML - Markdown conversion. It’s a bit slow and not very customizable but produces clean Markdown that preserves structure, links, and basic formatting. Best for content where preserving the document structure is important.</p><pre class="language-bash" data-lang="bash" v-pre=""><code class="lang-bash language-bash"><span class="token function">find</span> <span class="token builtin class-name">.</span> <span class="token parameter variable">-name</span> <span class="token string">'*.html'</span> <span class="token parameter variable">-exec</span> npx <span class="token parameter variable">--package</span> defuddle-cli <span class="token parameter variable">-y</span> defuddle parse <span class="token punctuation">{</span><span class="token punctuation">}</span> <span class="token parameter variable">--md</span> <span class="token parameter variable">-o</span> <span class="token punctuation">{</span><span class="token punctuation">}</span>.md <span class="token punctuation">\</span><span class="token punctuation">;</span></code><button class="docsify-copy-code-button"><span class="label">Copy to clipboard</span><span aria-hidden="hidden" class="error">Error</span><span aria-hidden="hidden" class="success">Copied</span><span aria-live="polite"></span></button></pre><ul><li><code>find . -name '*.html'</code>: Finds all HTML files in the current directory and subdirectories</li><li><code>-exec ... \;</code>: Executes the following command for each file found</li><li><code>npx --package defuddle-cli -y</code>: Installs and runs defuddle-cli without prompting</li><li><code>defuddle parse {} --md</code>: Parses the HTML file (represented by <code>{}</code>) and converts to markdown</li><li><code>-o {}.md</code>: Outputs to a file with the original name plus .md extension</li></ul><h3 id="pandoc"><a class="anchor" data-id="pandoc" href="#/convert-html-to-markdown?id=pandoc"><span>Pandoc</span></a></h3><p><a href="https://pandoc.org/" rel="noopener" target="_blank">Pandoc</a> is a bit slow and highly customizable, preserving almost all formatting elements, leading to verbose markdown. Best for academic or documentation conversion where precision matters.</p><p>Pandoc can convert from many other formats (such as Word, PDF, LaTeX, etc.) to Markdown and vice versa, making it one of most popular and versatele document convertors.</p><p><a href="https://youtu.be/HPSK7q13-40" rel="noopener" target="_blank"><img alt="How to Convert a Word Document to Markdown for Free using Pandoc Lynx renders the HTML as it would appear in a text browser, preserving basic structure but losing complex formatting. Best for quick content extraction or when processing large numbers of files.</p><pre class="language-bash" data-lang="bash" v-pre=""><code class="lang-bash language-bash"><span class="token function">find</span> <span class="token builtin class-name">.</span> <span class="token parameter variable">-type</span> f <span class="token parameter variable">-name</span> <span class="token string">'*.html'</span> <span class="token parameter variable">-exec</span> <span class="token function">sh</span> <span class="token parameter variable">-c</span> <span class="token string">'for f; do lynx -dump -nolist "$f" &gt; "${f%.html}.txt"; done'</span> _ <span class="token punctuation">{</span><span class="token punctuation">}</span> +</code><button class="docsify-copy-code-button"><span class="label">Copy to clipboard</span><span aria-hidden="hidden" class="error">Error</span><span aria-hidden="hidden" class="success">Copied</span><span aria-live="polite"></span></button></pre><ul><li><code>find . -type f -name '*.html'</code>: Finds all HTML files in the current directory and subdirectories</li><li><code>-exec sh -c '...' _ {} +</code>: Executes a shell command with batched files for efficiency</li><li><code>for f; do ... done</code>: Loops through each file in the batch</li><li><code>lynx -dump -nolist "$f"</code>: Uses the lynx text browser to render HTML as plain text<ul><li><code>-dump</code>: Output the rendered page to stdout</li><li><code>-nolist</code>: Don’t include the list of links at the end</li></ul></li><li><code>&gt; "${f%.html}.txt"</code>: Save output to a .txt file with the same base name</li></ul><h3 id="w3m"><a class="anchor" data-id="w3m" href="#/convert-html-to-markdown?id=w3m"><span>w3m</span></a></h3><p><a href="https://w3m.sourceforge.net/" rel="noopener" target="_blank">w3m</a> is very slow processing with minimal formatting. w3m tends to be more thorough in its rendering than lynx but takes considerably longer. It supports basic JavaScript processing, making it better at handling modern websites with dynamic content. Best for cases where you need slightly better rendering than lynx, particularly for complex layouts and tables, and when some JavaScript processing is beneficial.</p><pre class="language-bash" data-lang="bash" v-pre=""><code class="lang-bash language-bash"><span class="token function">find</span> <span class="token builtin class-name">.</span> <span class="token parameter variable">-type</span> f <span class="token parameter variable">-name</span> <span class="token string">'*.html'</span> <span class="token punctuation">\</span>

approach</li><li><strong>Need precise, complete conversion?</strong> Use pandoc</li><li><strong>Need a balance of structure and cleanliness?</strong> Try defuddle-cli</li><li><strong>Working with complex tables?</strong> w3m might render them better</li></ul><p>Remember that the best approach depends on your specific use case, volume of files, and how you intend to use the converted text.</p><h3 id="combined-crawling-and-conversion"><a class="anchor" data-id="combined-crawling-and-conversion" href="#/convert-html-to-markdown?id=combined-crawling-and-conversion"><span>Combined href="#/convert-html-to-markdown?id=crawl4ai">Crawl4AI</a> or <a href="#/convert-html-to-markdown?id=markdown-crawler">markdown-crawler</a>.</p><ol><li><strong>For research/data collection</strong>: Use a specialized crawler (like Crawl4AI) with post-processing conversion</li><li><strong>For simple website archiving</strong>: Markdown-crawler provides a convenient all-in-one solution</li><li><strong>For high-quality conversion</strong>: Use wget/wget2 for crawling followed by pandoc for conversion</li><li><strong>For maximum speed</strong>: Combine wget with lynx in a pipeline</li></ol><h3 id="crawl4ai"><a class="anchor" data-id="crawl4ai" href="#/convert-html-to-markdown?id=crawl4ai"><span>Crawl4AI</span></a></h3><p><a href="https://github.com/crawl4ai/crawl4ai" Crawl4AI is optimized for AI training data extraction, focusing on clean, structured content rather than complete site preservation. It excels at removing boilerplate content and preserving the main article text.</p><pre class="language-bash" data-lang="bash" v-pre=""><code class="lang-bash language-bash">uv venv
